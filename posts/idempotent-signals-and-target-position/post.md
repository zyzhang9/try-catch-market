# 为什么交易信号需要是幂等的

刚开始的时候，我看着他们把很多事情都先做成动作。

很多交易系统最开始都不会先长出一个清晰的 `executor`。先有的是策略。策略读行情，算 `fair price`，算 `exposure`，看账户状态，然后在某个时刻决定该不该动。真正先跑起来的，通常也不是一个“目标执行器”，而是一串很具体的动作：下单、撤单、改单、补一笔 `hedge`，再看结果。

所以系统一开始看起来并没有什么问题。

策略负责判断。

执行路径负责把判断落到交易所。

接口上流动的，也自然就是 `OrderPlacement`、`OrderCancelation` 这一类动作对象。

——

后来边界开始变得不舒服了。

因为策略真正想表达的，往往不是“现在去下一张单”。它真正想表达的更像是：仓位要变成多少，这条 `spread` 的净暴露要回到哪里，`inventory` 要从现在这个状态收敛到哪里。

动作只是把这个意图临时翻译成交易所语言的中间产物。可如果系统边界上传递的就是这个中间产物，那么一旦发生重试、重放、延迟确认、状态回补，策略原本只表达过一次的意图，就很容易在执行层被放大成两次、三次，甚至更多次。

这也是为什么，交易信号一旦处在策略和执行器的边界上，它最好是幂等的。

不是因为幂等听起来更“工程正确”。

而是因为在交易系统里，重复不是异常，重复本身就是现实。消息会重投，连接会抖，撤单确认会晚到，账户状态会和本地 `pending state` 暂时不一致。进程重启之后，系统还会试图根据本地状态和交易所状态重新接上之前那一段执行过程。

如果边界上传递的是动作型信号，那么重复一次，就多一次副作用。

在别的系统里，这可能只是日志多一行、任务多跑一遍。在交易系统里，多出来的是仓位。而仓位一旦被重复动作推偏，后面很多东西都会一起失真。风控看到的是错的，`PnL` 归因看到的是错的，策略自己看到的 `exposure` 也是错的。

——

我开始意识到，真正值得被单独对待的，不是信号格式，而是信号语义。

如果信号表达的是动作，那么它天然就是一次性的。

`buy 1`

被执行两次，结果一定和执行一次不同。

“撤掉这张单”被执行两次，也并不总是有稳定语义。第一次是真的撤单，第二次可能已经找不到对象，也可能打到了一个已经被替换过的本地状态。

但如果信号表达的是目标状态，事情就完全变了。

边界上不再传：

`buy 1`

而是传：

`target position = +1`

这时同一个信号被执行一次，或者被重复执行三次，语义上都还是同一件事。执行器需要做的不是“再次买入”，而是重新回答同一个问题：当前仓位离目标仓位还差多少。如果还差，就继续执行；如果已经到了，就什么也不做。

也就是说，幂等信号真正带来的，不只是去重能力，而是把边界从“命令边界”变成了“状态边界”。

——

等这个边界真的长出来以后，`executor` 才开始像一个独立层。

它不再只是一个命令消费者，而更像一个持续收敛目标的状态机。策略给它的不是瞬时动作，而是某个目标：`BTC target = +3`、`ETH target = 0`、`spread target = -200k notional`。然后 `executor` 自己去决定怎么把当前状态逼近过去。

它可以挂 `maker`，也可以打 `taker`，可以先撤掉冲突订单，可以等一个确认回来再继续，也可以在失败之后重新尝试。所有这些动作都仍然会发生，但它们不再是边界上的一等公民，而只是执行器内部为实现目标而产生的过程。

这时候一个很重要的东西才会自然长出来：跨越策略层级的执行进度。

如果边界上传递的是动作，你其实很难谈统一的 `progress`。因为“买一张单”没有天然的完成度。它要么发出去了，要么没发出去；要么成交了，要么没成交。每个策略都在用自己的方式理解这件事，于是进度语义被困在策略内部。

但如果边界上传递的是 `target position`，那么 `executor` 就第一次拥有了一个稳定的、可跨策略复用的进度定义。它可以直接展示：当前目标仓位是多少，当前实际仓位是多少，已经挂出去但尚未成交的 `working orders` 有多少，剩余 `gap` 还有多少，这次执行现在卡在挂单、成交、撤单还是状态确认。

——

看上去这只是一个防重问题，后来才会发现，它其实指向更深的一层：交易系统到底是把策略看成一连串动作的来源，还是把策略看成目标状态的来源。

如果是前者，那么执行层永远更接近一个带节流和重试的下单通道。

如果是后者，那么一个真正意义上的 `executor` 才会开始成形。它接受目标，它管理过程，它吸收重复，它暴露进度。而策略终于可以退回到它本来就更应该待的位置上：负责决定系统想去哪里，而不是亲自处理每一步该怎么走。
