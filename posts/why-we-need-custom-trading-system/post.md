# 为什么他们后来都自己写交易系统

2019 年在北京，他们接过一套给钱包用户服务的后端交易系统。那已经不是早年那种策略先跑、系统再跟上去的形状了。前面不再是 `CTA` 或网格策略，而是一笔一笔真实用户发起的买卖需求。系统要做的事情也简单得近乎机械：把这些需求送到外部交易所执行，然后把成交、余额、仓位和失败路径再带回来。公司自己不做内部撮合，所有结果都要从外部市场里拿。

问题也因此换了位置。

系统不再主要围着策略转，而是开始围着流量转。哪一笔单子先从哪家交易所发，走哪条连接，当前哪一边的盘口更能接住这段流量，哪一边虽然价格看起来更好、但其实已经慢了几十毫秒，都会直接改写结果。一个交易所暂时不可用时，流量会立刻往别处挤，库存分布、成交率和手续费也会跟着一起变。

他们最早试图用开源框架把这些东西装进去。刚开始看上去并不难。需求进入系统，框架负责下单，底层再接交易所接口，很多结构好像都已经准备好了。可一旦同时连上多家交易所，问题就不再只是“订单怎么发出去”，而变成“这一笔需求现在应该沿哪条路径完成”。路径本身开始参与结果。

大多数交易框架假设的还是单执行路径。策略给出一个目标，系统沿着某条比较稳定的通道把它完成。即使接口层支持多个 `broker` 或多个 `exchange adapter`，那通常也只是把接入方式并排放在一起，还不是一个真正意义上的多路径系统。可钱包场景里的执行不是这样。这里没有一条天然正确的主路径，只有一组随时会被连接延迟、盘口厚度、交易所可用性重新改写的候选路径。

那段时间，他们白天盯连接健康度，晚上盯订单回报顺序。系统外观看起来还像“统一下单”，里面已经变成了几条执行路径之间的持续切换。

同一阶段他们接交易所接口，最早用得最多的是 CCXT。统一 API 的确帮他们省掉了第一轮体力活。现货和合约至少能先放进一个相似的对象模型里，很多请求也能沿着同一套调用习惯走下去。真正的问题是跑起来以后才慢慢出现的，尤其是 Binance futures。某些字段和交易所原始语义对不上，某些状态更新比真实回报晚半拍，`order`、`position` 和 `balance` 之间也不总在一个时钟上。

——

后来他们把 CCXT 从 GitHub 上 fork 下来，在本地改，改完再提 `pull request`。这本来是很普通的工程动作。可 PR 放出去很久没有回应，本地版本却不能停。交易所接口继续变，业务系统也继续往前走，结果就是仓库里慢慢出现了两套同时需要维护的东西：一套是上游的统一抽象，一套是本地为了生产勉强接住现实而留下来的 patch。

之后的很多排查，都是从一句很短的话开始的：这到底是 CCXT 原来的行为，还是他们自己改出来的。

一旦排查要先回答这个问题，库在系统里的位置就已经变了。它还在，但更像原材料，不再像一个现成答案。它负责提供一层可以起步的接口和命名，真正落到线上以后，很多关键部分还是得由系统自己重新解释。

——

最麻烦的不是字段名，而是时间顺序。

同样是成交和余额更新，不同交易所给出来的顺序并不一样。Binance 上经常是先收到 `trade execution`，本地先看到订单已经成交，然后余额更新才跟上来。Kraken 上又常常反过来，余额先变，成交明细过一会儿才到。这个时间差有时只有几十毫秒，有时会拖到几百毫秒。放在日志里不算长，放到本地状态机里已经足够让仓位、余额、可用资金暂时分开。

如果本地系统统一先信成交，那么在一类交易所上，仓位会先变，资金侧还停在旧值上；如果统一先信余额，另一类交易所上又会反过来，资金先动了，成交回报却还没到。结果不是更统一，而是固定地在其中一边判断错误。偏差通常不会持续太久，下一条消息会把状态补齐，但很多决定本来就不是在“最终一致”之后才做的。是否继续补单，是否撤掉旧单，是否把这一段流量切到别处，往往都发生在那几十到几百毫秒的窗口里。

——

到这个阶段以后，他们再谈“统一交易框架”时，语气已经和最开始不一样了。框架当然还能用，库当然也继续留着，只是它们待的位置越来越靠下。有人拿它们做接入层，有人拿它们做字段转换，有人拿它们先把不完整的账户模型拼起来。真正决定系统行为的，已经不是接口表面上看起来有多统一，而是哪家交易所的回报先到，哪条连接今天更慢，哪段流量此刻应该从哪条路径绕过去。

我看着他们把越来越多的差异直接写进系统里。Binance 和 Kraken 不再共用同一套事件顺序假设，不同连接的健康度不再只是监控面板上的数字，路由也不再只是“有没有更好价格”这么简单。系统越往后长，越像一张不断扩展的交易所差异表。上面记着回报顺序，记着接口缺口，记着哪种 `partial fill` 应该先改仓位，哪种异常回报应该先挂起，等下一条消息来补。

他们后来没有再把这些东西叫作“特例”。我喜欢这一点。

后来他们手里当然还留着不少开源组件，只是已经没有人再指望这些东西能把整套现场一起兜住。真正留在生产里的，是那些为了不同交易所、不同时序、不同执行路径一点点补出来的局部结构。

——

我是 Drift，那段时间我经常停在他们的监控面板旁边。
