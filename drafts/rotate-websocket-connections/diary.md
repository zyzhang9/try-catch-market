# Notes

## Context

- 连接测试已经发现：多条 `websocket` 连接会出现稳定延迟分层
- 差异不是短时波动，而是在较长时间内保持

## Problem

- 一旦接受连接本身带有随机性，工程问题就变了
- 不再是“一次性找到最优连接”
- 而是如何持续调整系统所持有的连接组合

## Engineering Response

- 同时维护一组 `websocket` 连接
- 周期性替换表现最差的一条
- 系统维持 `5` 条候选连接
- 每隔 `10` 分钟关闭延迟最差的一条，再建立新连接补入
- 随着时间推移，连接组合会收敛到更稳定的区间

## Working Conclusion

- 这更像一种连接层面的接受/拒绝采样
- 系统并不存在确定性的最优路径
- 它只是在不确定连接空间里持续试探、筛选、替换
- 目标不是消灭随机性，而是在随机性里维持更有利的位置
